### 关于 #963 的调整说明（重要）
主仓库对 #963 的最终修复**不是**复用抢票间隔，而是新增独立参数 `rate_limit_delay_ms`（默认 **100ms**），429 后冷却该时长再 `continue`。
- 为满足「与主仓库对齐」，已采用此机制。
- ⚠️ **行为差异**：100ms 比你原来的 `--interval`（默认 1000ms）更激进。若想恢复你原先更稳的策略，把 `--rate-limit-delay-ms` 设为等于 `--interval` 即可（例如 `--rate-limit-delay-ms 1000`）。

## 四、与主仓库刻意保留的不同点

1. **无数据库 / 无 Web**：配置只来自 JSON 文件或 `--tickets-info`，不读 `ConfigDB`，`BuyConfig` 不含 `db=` 映射。
2. **无子进程/新终端逻辑**：删去主仓库 `start_new_terminal`、父进程 watchdog、`BTB_*` 环境变量协议（CLI 单进程直跑）。
3. **显示接入方式不同**：主仓库在 `app_cmd/buy.py` 用 `loguru_config` + `LOG_DIR`；本模块在 `main.py` 内联：Windows tty 默认走 Textual TUI，`--no-tui`/非 tty/缺依赖时回退纯文本 loguru。
4. **日志落盘**：固定写到 `btb_cli/btb_logs/app.log`（10MB 轮转、保留 7 天），不走主仓库的 UUID 日志名机制。
5. `interface/project.py` 为精简版，仅保留 `fetch_project_payload`。

## 五、使用说明

### 安装
```bash
pip install -r requirements.txt
```

### 最简启动
```bash
python main.py --config-file 订单.json
```

### 常用示例
```bash
# 定时开抢
python main.py --config-file 订单.json --time-start 2026-06-28T20:00:00

# 代理池 + 失效自动补池
python main.py --config-file 订单.json \
  --https-proxys http://u:p@1.2.3.4:7890,http://u:p@5.6.7.8:7890 \
  --proxy-api-url "https://你的代理商/api/getProxy" --proxy-api-protocol http

# 更稳的 429 策略（恢复旧行为：429 冷却 = 抢票间隔）
python main.py --config-file 订单.json --interval 1000 --rate-limit-delay-ms 1000

# 纯文本输出（重定向 / 服务器 / 终端无法渲染 TUI）
python main.py --config-file 订单.json --no-tui --log-level debug

# Bark 推送
python main.py --config-file 订单.json --notifier-config.bark-token YOUR_TOKEN
```

### 新增 / 关键参数
| 参数 | 默认 | 说明 |
|---|---|---|
| `--rate-limit-delay-ms` | 100 | 429 后冷却毫秒数；**调大更稳**（建议等于 `--interval`） |
| `--refresh-interval-min-count` / `--max-count` | 10 / 30 | 循环内随机多少次 create 后主动复检项目详情；`max=0` 关闭 |
| `--proxy-api-url` | 空 | 代理池全失效时自动拉新代理的供应商 API |
| `--proxy-api-protocol` | http | `http` / `socks5` |
| `--proxy-api-request-count` | 0 | 每次拉取数量，0=跟随当前池大小 |
| `--no-tui` | 关 | 禁用 Textual TUI，改纯文本 |

### 显示效果
- **Windows + 交互式终端**：默认 Textual 全屏 TUI（顶部倒计时/代理/冷却状态面板 + 下方彩色合并日志），与主仓库 CLI 一致。
- **其他情况**：自动回退纯文本（带 ✓/✕/⚠/⇄ 图标与状态快照行）。
- 已在入口统一把 stdout/stderr 切到 UTF-8，避免 Windows GBK 控制台下中文/二维码/`✓` 等字符报 `UnicodeEncodeError`。
